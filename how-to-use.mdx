Project Notes is most effective when used consistently. This page walks through the recommended workflow for capturing notes, organizing projects, and keeping everything findable — whether you're returning to a project after a week away or searching for a decision made months ago.

## Organizing Your Projects

A little structure upfront saves a lot of searching later. Follow these practices to keep your Projects section tidy and useful:

1. Create one page per project under the Projects section
2. Use the Project Template as a starting point for every new project
3. Keep the Status field up to date (Active, On Hold, Complete)
4. Link related projects using internal doc links

## Capturing Notes Quickly

The best note is the one you actually write down. Aim to write first, organize later — getting information out of your head and onto the page is more important than perfect formatting in the moment.

- Paste raw notes into a project page immediately after a session or call
- Use the Meeting Notes template for any call with more than 2 people
- Tag action items with **TODO** so they're easy to grep

## Keeping the Archive Clean

An overflowing active projects list makes it hard to focus. Keep things tidy with a few simple habits:

- Move completed projects to Archive > Completed Projects once they wrap up
- Keep the Ideas page for unscoped thoughts — review it quarterly to promote, discard, or refine them
- Delete duplicates ruthlessly

<Note>
  There's no "right" way to use this. Start with the templates, adjust over time, and keep consistency over perfection.
</Note>

## Naming Conventions

Consistent names make search and browsing dramatically easier. Use these patterns across your notes:

| Type | Pattern | Example |
|---|---|---|
| Project page | Project name | Website Redesign |
| Meeting note | Date + topic | 2024-01-15 Kickoff |
| Research note | Topic area | Competitor Analysis |
