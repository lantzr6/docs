Use this template whenever you start a new project. Copy the raw markdown below into a new page under the Projects section, fill in your details, and delete any sections you don't need.

<Tip>
  To create a new project page, duplicate this template and rename it to match your project. Keep the section structure consistent — it makes projects easier to compare at a glance.
</Tip>

```markdown project-template.md
# Project Name

> Status: 🟢 Active | Start: YYYY-MM-DD | Target: YYYY-MM-DD | Owner: Your Name

One-sentence description of what this project is and why it matters.

## Goals

- Goal 1
- Goal 2
- Goal 3

## Open Tasks

- [ ] Task 1
- [ ] Task 2
- [ ] Task 3

## Decisions Log

| Date | Decision | Rationale |
|------|----------|-----------|
| YYYY-MM-DD | Decision description | Why this was chosen |

## Notes & Updates

### YYYY-MM-DD

- Update or note here

## Related Resources

- [Resource name](URL)
```

## Sections Explained

<AccordionGroup>
  <Accordion title="Goals">
    What the project needs to achieve. Keep this list short and measurable.
  </Accordion>
  <Accordion title="Open Tasks">
    Current to-do items. Cross off with `[x]` when done, or remove them.
  </Accordion>
  <Accordion title="Decisions Log">
    Record key decisions with the date and reasoning. Invaluable when revisiting projects.
  </Accordion>
  <Accordion title="Notes & Updates">
    Dated journal entries for meeting summaries, blockers, and progress.
  </Accordion>
  <Accordion title="Related Resources">
    Links to designs, docs, repos, and other reference material.
  </Accordion>
</AccordionGroup>
